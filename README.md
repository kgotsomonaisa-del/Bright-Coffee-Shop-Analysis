Bright Coffee Shop Sales Analysis – 
1.	Overview

1.1	The Bright Coffee Shop Sales Analysis project was designed to support a newly appointed CEO in making data-driven decisions to grow revenue and improve product performance. 

1.2	Historical transactional data from the coffee shop compiled from 1 January to 30 June 2023 was used for the project focusing on extracting meaningful insights that can guide business strategy.
1.3	Daily sales were analysed and data was transformed into actionable insights. This involved working with data processing tools using Databricks, performing calculations using Excel, and presenting findings using Canva in a clear and visually engaging way.
1.4	The analysis aimed to answer key business questions, including identifying the top-performing products, understanding peak sales periods, and evaluating sales trends across different time intervals and product categories. 
1.5	These insights will help the business optimize operations, improve customer experience, and increase profitability.
2.	Case study process and findings 
2.1	Raw data was analysed and instructions read from the project description document and the objectives and expected results identified. 

2.2	Data from the raw CSV data file was also inspected and analysed to verify what information we have (column names) to determine what information could be retrieved from the table.

2.3	Databricks was used to perform SQL codes to retrieve the following information:

2.3.1	All rows and columns from the table
2.3.2	The start and end date of capturing the data, 1 January – 30 June 2023.
2.3.3	Distinct store locations, product categories, product types and product details
2.3.4	Transaction dates
2.3.5	Minimum and maximum unit prices
2.3.6	Products generating the most/least revenue
2.3.7	Products sold across all store locations
2.3.8	Total revenue and revenue generated at different store locations
2.3.9	Transaction dates and times
2.3.10	Busy and/or slow business times
2.3.11	Revenue per day, week and/or month

2.4	Combining the codes above (2.3.1 to 2.3.11), a long SQL code was then performed to create one query which aggregated the initial CSV data file and enabled more insights to be drawn.
2.5	From the result of the query above, data was extracted and pivot tables created to visualise insights
2.6	Finally, a presentation to be used to convey insights from the captured data and recommendations for improvements in sales to the CEO was created.

3.	Tools used to complete the project 
The following tools were used to complete the project:
3.1	Miro for data planning & architecture showing how data flows from the source to analysis 
3.2	Databricks for data processing, cleaning and transformation using SQL coding
3.3	Microsoft Excel for analysing & visualising data, as well as creating dashboards and charts.
3.4	Canva for preparing the presentation and reporting insights and recommendations to the CEO.
